# 题库自动生成智能体开发方案

## 一、 项目定位
### 1. 核心目标
本项目旨在打造一个高智商、零延迟感知、无视外网反爬屏障的“题库自动生成智能体”。它能够根据用户输入的任意职业和考核知识点，结合实时检索的最前沿、最权威的互联网参考资料，快速且精准地生成符合规定题型的专业级试题。

### 2. 需求痛点
- **格式幻觉严重**：传统大模型在面对复杂的约束时容易“失控”，比如明确要求出“单选题”，模型却自行发散输出“判断题”或“简答题”。
- **知识库陈旧干瘪**：大模型的预训练数据存在截止时间，在生成诸如“2024最新行业规范”相关的案例题时，往往闭门造车、脱离现实工作场景。
- **高价值数据获取难（反爬虫）**：行业白皮书或顶级专业网站通常配有 Cloudflare 5秒盾等极强的反爬机制，普通工具难以触达。
- **响应过于缓慢**：要求模型生成带解析的试题所需耗时极长（几十秒），用户只能面对空白屏幕干等。

---

## 二、 技术选型
为了彻底解决上述痛点，系统采用以下全栈技术链路：
1. **工作流编排引擎**：Dify（作为核心中枢驱动）。
2. **底层大语言模型**：DeepSeek-V3 （提供顶级推理的“高智商大脑”）。
3. **内容检索聚合**：
   - 引流端：`Google Search API`（全球最强权威搜索）。
   - 抓取端：`Firecrawl`（专为 AI 开发的反防爬虫网页提取神兵）。
4. **前端交互技术**：React/Vue + SSE (Server-Sent Events) 长连接流式渲染。

---

## 三、 系统整体架构
系统采用“前后端分离 + 智能体 API 调度”的极简流式架构：
1. **控制台层 (Frontend)**：向外提供表单交互引擎与流式排版展示器。
2. **网关层 (Dify API)**：处理请求校验，并基于 `response_mode: streaming` 建立双向通信。
3. **编排引擎层 (Dify Workflow)**：内部采用“双轨并行”拓扑，分为“规则控制流”与“外网爬虫流”。
4. **模型推理层**：吸收双规流数据，执行推演输出。

---

## 四、 核心流程设计（闭环）
1. **【输入】**：用户提交考点维度参数（岗位、难度、题型、数量）。
2. **【分发】**：参数兵分两路并发。
3. **【防篡改规约】**：左侧逻辑用原生代码瞬间写出霸王条款，扼杀题型幻觉。
4. **【穿透爬取】**：右侧逻辑搜索最新外网地址 ➔ 绕过盾构机拿到纯文本 Markdown。
5. **【流式生成】**：上述所有资源就位进入大模型，大模型逐字吐出试卷。
6. **【输出闭环】**：前端秒级响应、渲染，用户审阅或带格式导出。

---

## 五、 核心流程详解
*   **0延迟约束机制**：传统校验是在大模型生成后进行验证，导致耗时翻倍。本方案采用“前置拦截”，通过毫秒级的 Python 代码在请求大模型前，就将题型彻底锁死在 Prompt 顶端。
*   **降维打击爬虫机制**：并非直接让大模型去阅读乱七八糟的几百条搜索摘要，而是通过 Google 提取到“最权威的那 1 个链接”，直接派出专门用于穿透验证码的 Firecrawl 爬虫。它能把对方完整的白皮书或文档脱壳为干干净净的纯净字符。这才是系统拥有深度高质量业务场景的关键。

---

## 六、 Dify 工作流设计
本核心基于 Dify 图形化 DAG 工作流搭建：
1. **`Start` 节点**：接收前端的 5 个变量。
2. **`Code` 节点（左轨-类型安全锁）**：将类似 `单选题` 的变量，转化为一段绝对命令：*“必须且只能输出单选题，绝对不允许出现判断题，违规将导致崩溃”*。
3. **`Google` 节点（右轨-检索引路）**：拼接职业和知识点，查出最新官网快照。
4. **`Code` 节点（右轨-提链脚本）**：用正则抓取 Google 结果中排名 TOP 1 的网址。
5. **`Firecrawl` 节点（右轨-反爬脱壳）**：传入提取的 URL，无视防御，生吞整站文本。
6. **`LLM` 节点（交汇出卷）**：`DeepSeek-V3` 吸收左右两路素材，开始发力。
7. **`End` 节点（流推）**：直通底层，连接流式事件发送。

---

## 七、 提示词设计（核心能力）
绝不依赖温和的交互语气，而必须使用极致严谨的结构化 Prompt：
```text
{{Code节点生成的霸王条款级格式约束}}

你是一位拥有20年实战经验的【{{职业}}】领域资深考官与终极培训专家。
请结合以下通过强制反爬拿到的全网页硬核资料（如适用）进行出题：
【深度原网页参考资料】：
{{Firecrawl爬虫抓取的长文}}

【绝对红线指标】：
1. 必须提供真实的行业业务场景作为题干背景，拒绝干瘪的纯理论问答，要求高度贴合实际工作。
2. 若为选择题，干扰选项必须具有极强的迷惑性（针对业内的真实常见坑点设计）。
3. 所有题目必须同时输出“深度解析”，逐一剖析错误选项的槽点。
4. 绝对禁止输出任何类似“好的”之类的客套废话，直接开始输出题目Markdown。
```

---

## 八、 系统界面设计
1. **出题操作区**：
   - “职业角色、考点说明”输入框（支持大文本块）。
   - “难度指数、题目类型、题目数量”下拉控制组件。
   - 带有 Loading 和 防连击动效的“智能出卷”核心按钮。
2. **答卷展板区**：
   - **占位态**：使用动感的骨架屏（Skeleton）在检索期间稳定用户焦躁情绪。
   - **渲染态**：使用流式 Markdown 渲染引擎（搭配 `highlight.js` 处理排版），使文字如瀑布流般打字呈现。
3. **交付配套区**：提供【一键复制】、【下载为 PDF】等考后教务功能。

---

## 九、 项目价值
- **效率跃升**：免除了漫长的人工出题周期。老师/管理者只需要写一句话，就能即时生成一套媲美真题的考卷。
- **专业度极值化**：依靠全球公网最新资料托底，题目的情境永远与当前前沿同步；依靠顶级 LLM 的逻辑深度，其解析能力甚至优于一般从业讲师。
- **零人工运维防爬**：以往的爬虫节点常常因对方网站升级防爬防火墙而瘫痪，采用基于商用级协议的反爬虫代理服务免除了昂贵的反爬运维成本。

---

## 十、 创新点
1. **防格式幻觉的最优解**：行业内首创的前置代码强校验替代了后置重试循环，实现了 0 网络损耗保证格式 100% 正确。
2. **信息漏斗的绝佳组合**：`Google广度定位` + `代码精准锁链` + `Firecrawl深度爆破` 这套组合拳，达到了企业检索增强（RAG）系统中极为罕见的高信噪比。
3. **全链路流体打通**：通过抛弃一切拖累大模型直出表现的繁杂后处理，做到了数千字附带深度解析的考题“无需干等、瞬间推流”，将前端体感做到了极致。

---

## 十一、 总结
本开发方案脱离了传统“知识图谱搜索+模型生成”的常规刻板路径。凭借极简的工作流并发设计、强烈的提示词约束以及强力反爬降维打击插件，在满足不同领域专业深度的前提下，为终端用户带来行云流水般的交互体验。它将成为新一代数字化教务与实战培训环节中最锋利的自动化利刃。
