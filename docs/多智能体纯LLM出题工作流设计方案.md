# Dify 多智能体纯 LLM 出题工作流设计方案

> 基于题库智能体系统文档，设计无插件、纯大模型+脚本的高质量自动出题工作流

## 一、设计原则

| 约束 | 实现方式 |
|------|----------|
| **禁用插件** | 不使用 Google Search、Firecrawl 等工具，全部由 LLM 内生知识推理 |
| **多模型分工** | R1 负责深度知识架构，V3 负责拆解与命题，各司其职 |
| **脚本约束** | Code 节点实现题型锁定、JSON 校验、数量兜底，防止大模型偏离用户需求 |
| **迭代出题** | 每次迭代仅输出 1 道题，按用户数量循环，确保质量可控 |
| **质量保障** | 前置约束 + 分步推理 + 剥洋葱式解析要求 |

## 三、节点详细规格

### 1. Start 节点
- **变量**: profession, topic, question_type, difficulty, question_count

### 2. Code: 题型约束锁
- **输入**: question_type
- **输出**: strict_rules
- **逻辑**: 根据题型生成"霸王条款级"格式约束，0 延迟阻止题型幻觉
  - 单选题 → 强制 A/B/C/D、唯一正解
  - 多选题 → 至少两正解
  - 判断题 → 仅正确/错误
  - 简答题 → 论述型，禁止选项

### 3. LLM1: 知识架构专家 (DeepSeek-R1)
- **角色**: 认知心理学 + 行业泰斗
- **输入**: profession, topic
- **输出**: 2500+ 字深度知识蓝图
- **职责**: 替代外网检索，内生推理生成「虚拟参考白皮书」
- **内容**: 底层逻辑、灰色地带、认知陷阱、实战颗粒度

### 4. LLM2: 维度拆解官 (DeepSeek-V3)
- **角色**: MECE 原则 + 布鲁姆分类法
- **输入**: blueprint, question_count
- **输出**: JSON 数组，N 个互不重叠的考核维度
- **格式**: `["维度1：场景A；考核点X", "维度2：场景B；考核点Y", ...]`
- **约束**: 输出纯 JSON，禁止解释文字

### 5. Code: JSON 解析与数量校验
- **输入**: 拆解官的 text 输出
- **输出**: list (array[string])
- **逻辑**:
  - 剥离 ```json 等 markdown 包裹
  - 解析 JSON 数组
  - 若数量 < question_count：用"综合考核"补齐
  - 若数量 > question_count：截取前 N 项
  - 解析失败时返回 `["综合考核：{{topic}}"]` 兜底

### 6. Iteration 节点
- **iterator_selector**: Code 解析器的 list
- **output_selector**: 单题命题专家的 text
- **变量传入**: strict_rules, blueprint, diff, question_type, profession, topic
- **内置变量**: item（当前维度字符串）, index

### 7. LLM3: 单题命题专家 (DeepSeek-V3) [迭代内]
- **每次迭代仅生成 1 道题**
- **输入**: rules, blueprint, task(item), diff, question_type, profession, topic
- **约束**:
  - 必须遵守 {{rules}} 题型
  - 必须基于当前 {{task}} 维度，不得发散
  - 剥洋葱式解析：命题立意 → 陷阱拆解 → 排除法 → 原理校准 → 进阶思考

### 8. Code: 结果聚合器（可选）
- **输入**: iteration 的 output 数组
- **输出**: 合并为最终 Markdown 文本
- **逻辑**: `"\n\n---\n\n".join(output_array)`

### 9. End 节点
- **输出**: iteration 的 output 或聚合后的 result

## 四、脚本约束详解（防大模型跑偏）

| 脚本 | 防什么问题 | 实现 |
|------|------------|------|
| 题型约束锁 | 要单选给判断、要简答给选择 | 将题型转为绝对命令式 Prompt 前缀 |
| JSON 解析 | 拆解官输出非 JSON 或格式错误 | 正则剥离 + try/except + 兜底单元素数组 |
| 数量校验 | 拆解出 3 个维度但用户要 5 题 | 不足补 "综合考核"，超出截断 |

---

## 五、质量保障机制

1. **前置约束**: Code 在调用 LLM 前写入红线指令
2. **分步推理**: 知识蓝图 → 维度拆解 → 单题生成，避免一次生成多题导致质量滑坡
3. **单题迭代**: 每轮只出 1 题，模型聚焦度高
4. **维度互斥**: 拆解官要求 MECE，避免题目重复
5. **剥洋葱解析**: 强制 5 层解析结构，提升专业度

## 六、模型选型建议

| 节点 | 推荐模型 | 理由 |
|------|----------|------|
| 知识架构专家 | DeepSeek-R1 | 深度推理，适合复杂知识推演 |
| 维度拆解官 | DeepSeek-V3 | 结构化输出强，温度低 |
| 单题命题专家 | DeepSeek-V3 | 平衡速度与质量，流式友好 |
