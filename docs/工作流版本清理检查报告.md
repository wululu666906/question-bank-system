# 工作流版本清理检查报告

## 一、使用情况分析

### 1. 前端 (frontend)
- **difyClient.ts**：通过 Dify API 调用工作流，**不直接引用任何 YAML 文件**
- 实际使用的工作流：由用户在 Dify 中导入的任意 DSL 决定

### 2. question_generator 模块
- **generate_dsl.py**：生成 `dify_workflow.yml`（外部 API 出题引擎）
- **dify_workflow.yml**：**正在使用**，作为「外部 API 智能出题引擎」的 DSL
- **api_server.py**：提供 Python API，供 Dify HTTP 节点调用

### 3. 文档推荐
- `智能题库工作流开发计划书_V2.md` 推荐：`question_factory_v2_full_featured.yml`（**当前仓库中不存在**）
- `多智能体纯LLM出题工作流设计方案.md` 推荐：`question_factory_multi_agent_pure_llm.yml`（**当前仓库中不存在**）
