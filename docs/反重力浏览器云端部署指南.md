# 如何使用反重力浏览器部署智能题库项目到云端

## 一、反重力浏览器说明

在本文档语境下，「反重力浏览器」有两种可能指代：

### 1. 项目中的辅助工具角色

根据 `ARCHITECTURE.md`，反重力浏览器在智能题库系统中的定位为**数据处理自动化工具**：

- 批量抓取资料（法规/题库）
- 自动整理文本
- 自动上传数据
- 批量导出题目

在此含义下，它主要用于**数据准备与整理**，不直接负责云端部署。

---

### 2. Google Antigravity（AI 编码与部署平台）

[Google Antigravity](https://antigravity.google/) 是 Google 推出的 **AI 代理式 IDE**，支持：

- 架构规划与代码生成
- 终端命令执行
- **构建并部署到 Google Cloud**（Cloud Run、GCS、BigQuery 等）
- 浏览器扩展：与 Chrome 配合完成 UI 测试、自动化操作

若你的目标是「用 AI 辅助将项目部署到云端」，**Google Antigravity** 是最契合的选择。

---

## 二、项目云端部署总体思路

智能题库项目主要包括：

| 组件 | 说明 | 部署方式 |
|------|------|----------|
| **Dify 工作流** | 出题引擎（DSL 文件） | 导入至 Dify Cloud 或自建 Dify |
| **前端** | React + Vite（题库交互界面） | Vercel / Cloud Run / 静态托管 |
| **Dify 平台** | 若需自托管 | GCP / 阿里云 / 腾讯云等 |

---

## 三、方案 A：使用 Google Antigravity 部署到 Google Cloud

### 3.1 前置准备

1. **安装 Google Antigravity**
   - 访问 [antigravity.google/download](https://antigravity.google/download) 下载桌面端
   - 或安装 VS Code 扩展
   - 使用 Google 账号登录

2. **配置 Google Cloud**
   - 创建 GCP 项目并开通计费
   - 安装并配置 [gcloud CLI](https://cloud.google.com/sdk/docs/install)

3. **在 Antigravity 中打开项目**
   - 使用「打开文件夹」指向你的题库项目目录（如 `c:\Users\Auraa\Desktop\题库`）

### 3.2 让 Antigravity 协助规划与部署

在 Antigravity 的 **Agent Manager / Playground** 中，可输入类似 prompt 让其生成部署方案与脚本：

```
我需要将「智能题库」项目部署到 Google Cloud。项目结构如下：
- frontend/：React + Vite 前端，需构建为静态站点
- 工作流 DSL：question_factory_v2_full_featured.yml，需导入 Dify

请帮我：
1. 编写 Dockerfile，将前端构建并部署到 Cloud Run（或建议更合适的托管方式）
2. 生成一键部署脚本（setup.sh），包括：启用所需 API、创建 GCS 存储桶（如有）、部署 Cloud Run 服务
3. 配置环境变量占位符（如 DIFY_API_URL、DIFY_API_KEY），方便后续填入真实值
```

### 3.3 设置审阅策略

在 Antigravity 右上角设置中建议：

- **Review Policy** → `Asks for Review`（每步需人工确认）
- **Terminal Command Auto Execution** → `Request Review`（终端命令需批准后再执行）

这样可在执行前检查脚本和命令是否安全、符合预期。

### 3.4 执行与验证

1. 审阅生成的 `setup.sh`、`Dockerfile`、`cloudbuild.yaml` 等
2. 按提示批准终端命令，完成资源创建与部署
3. 在 Cloud Run 控制台查看服务状态，并用 Walkthrough 验证部署结果

### 3.5 前端对接 Dify

部署完成后，在 Dify 中：

1. 导入 `question_factory_v2_full_featured.yml` 创建工作流
2. 在「应用」-「API 访问」中获取 API Key 和 Base URL
3. 将 `DIFY_API_URL`、`DIFY_APP_ID`、`DIFY_API_KEY` 配置到前端环境变量
4. 前端通过 Dify OpenAPI 调用出题工作流

## 四、方案 B：快速部署（Dify Cloud + Vercel）

不依赖 Antigravity 时，可按以下步骤完成部署。

### 4.1 Dify 工作流

1. 登录 [Dify Cloud](https://cloud.dify.ai) 或自建 Dify
2. 新建「工作流」应用
3. 导入 DSL：`question_factory_v2_full_featured.yml`
4. 在应用设置中获取 **API Key** 和 **Base URL**

### 4.2 前端部署到 Vercel

1. 将前端代码推送到 GitHub
2. 在 [Vercel](https://vercel.com) 中导入该仓库
3. 构建设置：
   - Framework Preset: `Vite`
   - Build Command: `npm run build`
   - Output Directory: `dist`
4. 添加环境变量：
   - `VITE_DIFY_API_URL`：Dify API 地址
   - `VITE_DIFY_APP_ID`：应用 ID（如使用）
   - `VITE_DIFY_API_KEY`：API Key

### 4.3 反重力浏览器（自动化辅助）

反重力浏览器可作为**自动化辅助**：

- 批量抓取法规、题库等原始资料
- 整理成符合 Dify 知识库或工作流输入格式的文本
- 通过浏览器自动上传到 Dify 知识库或触发工作流

这类操作通常配合 Selenium、Puppeteer 或专用自动化工具实现，与云端部署本身是并行环节。

## 五、方案 C：自托管全栈（云服务器）

如需在单台云服务器上自托管 Dify + 前端：

### 5.1 服务器建议

- 阿里云 ECS / 腾讯云 CVM / GCP Compute Engine
- 配置：2 核 4G 起，Ubuntu 22.04

### 5.2 部署 Dify

```bash
# 克隆 Dify 官方仓库
git clone https://github.com/langgenius/dify.git
cd dify/docker

# 按文档配置 .env
cp .env.example .env
# 编辑 .env，配置数据库、Redis、模型 API 等

# 启动
docker compose up -d
```

### 5.3 部署前端

```bash
cd frontend
npm install
npm run build

# 使用 nginx 托管 dist/
# 或使用 Docker 多阶段构建
```

### 5.4 反重力浏览器辅助

在本地或云端 Windows/Linux 环境中运行反重力浏览器，用于：

- 批量抓取与整理资料
- 通过 API 或浏览器自动化，将数据导入 Dify 知识库
- 批量导出题目到 Excel / 数据库
