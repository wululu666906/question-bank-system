# Dify 智能题库生成引擎（Google+Firecrawl 穿透防爬版）开发文档

## 1. 业务愿景与系统定位
本套工作流 (Workflow) 旨在为全行业/全职业培训与考核系统提供**高度自动化、极低延迟感知、无视外网反爬策略**的高智商试题生成器。
其核心特性在于：首字流式零延迟（Typewriter 效果）、完全摒弃大模型随意发散题型的幻觉、且具备极强的专业网页溯源与正文提取能力。

## 2. 节点架构总览 (并联双轨处理)

为了追求极致并发，该工作流被设计为**无阻塞双轨结构**：

```text
                 [Start (用户输入参数)]
                       |
        +--------------+--------------+
        | (轨道一：逻辑锁)            | (轨道二：数据采掘网)
        ↓                             ↓
 [Code: 类型安全锁]             [Tool: 谷歌搜索 (Google Search)]
 (0毫秒, 生成红线级系统指令)             | 
                                      ↓ 
                            [Code: 智能 URL 链提取]
                            (从谷歌结果提炼全球第一名网页外链)
                                      |
                                      ↓
                            [Tool: Firecrawl 反爬脱壳抓取]
                            (穿透五秒盾/JS渲染，榨出纯净 Markdown 原文)
        |                             |
        +--------------+--------------+
                       ↓
         [LLM: 顶级命题大师 (DeepSeek-V3)]
         (融合硬核指令与全网真实案例的长文本，流式出卷)
                       |
                       ↓
               [End: 前台流式直连]
```

## 3. 各节点详细技术逻辑说明

### 3.1 Start (节点: 接收表单)
- **profession (String)**: 用户角色/行业（如：架构师、法务专家）。
- **topic (String)**: 考察的细分考点。
- **question_type (Select)**: 支持选择题、多选题、判断题、简答题等枚举。
- **difficulty (Select)**: 动态指示模型心智的认知层级指标。
- **question_count (String)**: 生成数量。

### 3.2 Code 节点（类型防御器）
- **功能**：通过纯 Python 原生脚本，针对 `question_type` 执行 `if-else` 输出“霸王条款级”约束指令 `{{strict_rules}}`。
- **优势**：0网络请求、0延迟，将大模型最容易犯的“不遵守格式/题型错误”锁死在萌芽阶段，完美捍卫数据可用度。

### 3.3 Google Search（强劲引路人）
- **功能**：作为“寻找金矿”的第一步，对参数动态拼凑：`{{profession}} {{topic}} 核心考点 案例 实操标准` 进行全球最强引流。
- **注意**：需要您在 Dify 工具库填入 Google 搜索引擎 API Keys。

### 3.4 Code 节点（链接猎手）
- **功能**：专门对付 Google 传回的长串乱杂数据，通过正则表达式 `re.search(r'https?://[^\s]+', text)` 提取第一条纯净有效的 URL。
- **输出**：一个名为 `target_url` 的干净外链变量，直接喂给爬虫系统。

### 3.5 Firecrawl Scrape（降维打击爬虫）
- **功能**：通过专门针对 AI 和大语言模型设计的爬虫引擎，对上游锁定的 URL 发起访问。
- **核心价值**：它可以突破普通的 `urllib`/`requests` 会遭遇的 `403 Forbidden`（Cloudflare 拦截），将花里胡哨的前端页面强行剥离渲染，提纯出干干净净的长文本 Markdown。这也是高质量考题不脱离“现代真实案例”的绝对功臣。

### 3.6 LLM 核心发散大脑
- **选型**：推荐拉满推理心智的模型（如 DeepSeek-V3 / GPT-4o）。
- **流批控制**：Dify 环境下，由于这是最后一个处理文本的节点，只要在 API 调用阶段指明 `response_mode: streaming`，所有出题文字能直接像雪花一样落到前端，用户无需傻等30秒！。
- **Prompt 设计哲学**：严禁模型输出客套废话，强制基于 Firecrawl 给出的顶级真实现状作为题干背景设计。

## 4. 后续维护与迭代方向
1. **备用兜底机制**：目前 `智能链提取` 代码如果搜不到链接，给了维基百科作为兜底。可按需改成内部企业知识库链接。
2. **插件平替**：若后续因费用问题暂停 Google/Firecrawl，可在画布左侧随时将其删除，直接将原流线断开处连接至 `Tavily` 或 `Wikipedia` 等开源替代方案，不影响主体。
