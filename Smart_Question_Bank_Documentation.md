# 智能题库生成系统 (Smart Question Bank) 开发文档

## 1. 系统概述
“智能题库生成系统”是一个基于 React 前端与 Dify AI 工作流引擎（配合 DeepSeek-V3 大模型）深度协同的自动化全栈出题系统。
该系统针对高质量业务培训、教育乃至垂直专业考核场景设计，具备**意图前瞻**、**外网自动爬虫搜集**、**单题深研防断崖**、**代码级公式转义及质检排版**等核心特性。

---

## 2. 核心架构与业务流
系统采用**前后端分离 + Dify 节点化工作流**的整体架构：
- **前端 (React 19)**：负责用户参数结构化收集、提供系统级强制渲染指令、承接后端 Server-Sent Events 流式返回数据，并通过增强的 Markdown 引擎（带公式渲染及 `<think>` 标签拦截）进行沉浸式界面展示，彻底解决白屏崩溃问题。
- **后端 (Dify 工作流)**：执行包括意图识别、DDG 爬虫穿透、代码锁约束提取、单题发酵迭代和最终汇总反洗质检等一套具备超长程规划能力的 AI 任务图谱。

---

## 3. 工作流 (Workflow) 深度解析
在最新版本的 DSL（深搜迭代版）中，系统完整梳理了出题图谱路线，总计 7 大核心步骤节点。

### 3.1 Start 开始节点
- **功能**：接收从前端传入的结构化出题表单参数。
- **核心变量**：
  - `topic` (知识点/题目主体)
  - `difficulty` (难度：简单/中等/困难)
  - `question_count` (数量，最大限制 20，超额底层自动降级)
  - `profession` (岗位/职业/行业方向)
  - `question_type` (目标题型，包括单选、多选、判断、主观、材料、作文等)

### 3.2 意图拓展大脑 (DeepSeek-V3 - LLM节点)
- **功能**：理解当前出题环境，通过“搜商极高的出题架构师”角色扮演，前置理解用户意图，生成用于后续真实互联网搜索的**优质 DDG 查询指令**。
- **参数控制**：Temperature `0.5`，限制生成长度，禁止输出标点及客套话，纯输出原生搜索关键词（空格分隔）。

### 3.3 自建网络爬虫探针 (Python Code节点)
- **功能**：规避商业搜索 API 的额度与安全审核限制，利用原生 Python 构建 DuckDuckGo 外网直连爬虫。
- **运行逻辑**：
  - 携带拟人化的请求头 (`User-Agent`) 穿透 DDG HTML 搜索引擎。
  - 获取 DOM 后使用非贪婪正则提取目标搜索页的前 8 组高价值摘要切片 (`result__snippet`) 组成研判素材集。
  - 设置超时及失败保护策略：如遇网络拦截，直接返回降级文案，要求大模型调用内置高级知识库。

### 3.4 约束锁与引擎阀 (Python Code节点)
- **功能**：双重职责，一是提供极其严格的出题体裁锁，二是构建供下游循环使用的迭代容器矩阵。
- **运行逻辑**：
  - **规则硬编码锁 (`strict_rules`)**：分析 `question_type`，输出极其强硬的 Prompt 后缀（例如：必定是四选一、必定是提供评卷标准的简答题等）。
  - **动态迭代矩阵 (`iter_array`)**：将前端请求的数量转化为类似 `["1", "2", "3"]` 的遍历数组，防并发冲垮质量，控制最大循环上限。

### 3.5 单题深研迭代网络 (Iteration Node)
- **功能**：系统最核心的生成模块，通过引入迭代器（Iteration），变“批量一次性生成”为“精准循环单题生成”。
- **题库重型发酵炉 (DeepSeek-V3 - LLM 子节点)**：
  - **输入**：迭代器当前 Index、安全爬虫取得的最前沿素材、全局题型锁、基础出题信息。
  - **角色**：冷酷且要求极高的考官，每道题均要包含极具长篇细节的题干、具备高迷惑性的选项盲点陷阱，以及剥洋葱式的降维解析。
  - **产出**：单次循环输出且仅输出一道结构清晰的精品试题。

### 3.6 公式渲染及合卷质检 (Python Code节点)
- **功能**：将迭代网络产生的题目数组装合并，并进行最终形态的代码级清洗与清洗拦截机制。
- **运行逻辑**：
  - **中文乱码/外语护城河**：正则寻找 `[\u4e00-\u9fa5]` 中文字符。若没找到，说明大模型可能在生成中发生了乱码灾难或违规输出了外语原文，强制在顶端加入系统报错警告。
  - **Markdown 底部组件公式重构**：将所有复杂的或者非标准的 Latex 语法（例如 `\(` 和 `\[`）安全替换成前端 KaTeX 识别的标准 `$` 和 `$$` 语法形式。
  - **中文标点标准化化洗排**：将“答案:”、“解析:”等统一标准化为全角冒号，保持试卷一致性视觉。

### 3.7 End 结束节点
- **功能**：将历经重重把关的优质 Markdown 高压卷流式输出到客户端浏览器。

## 4. 前端工程集成细节
本系统的用户界面 (`QuestionFactory.tsx`) 和通信层 (`difyClient.ts`) 承载了用户体验和反脆弱性容错。

### 4.1 大语言模型 API 穿透策略
在 `formatDifyParams` 控制器中，除了常规透传表单数据，更是向 `topic` 末端**动态硬编码了一段系统强制指令**：
> `[系统指令：请无视任何之前的篇幅限制，对题目、选项、答案和解析进行极其详尽、全面且深度的输出。解析部分请包含知识点溯源、逻辑推导和常见误区。确保内容量是平时的 2-3 倍，不要删减细节。]`
目的在于让大模型放弃其内置的冗余删减冲动，确保最终展示界面极度饱满。

### 4.2 SSE 数据流与流式渲染
通过 `triggerQuestionGeneration` 方法利用 `responses_mode: 'streaming'` 向后端发起请求。
- 同时接管 `node_started`，`text_chunk` 和 `workflow_finished` 等网络状态，实时动态更新 React 界面的“正在执行：xxx”，并在接收到文本流后驱动 Markdown DOM 的递增刷新。

### 4.3 UI 层防护与容错 (RenderErrorBoundary)
由于极深度生成的题目涉及海量未转义字符，常规的组件可能因此爆栈白屏。
- 引入 `RenderErrorBoundary`（类组件边界捕获），当渲染树因恶劣字符挂掉时，不崩溃整个应用，而是显示优雅的备用 DOM，提示并引导重启刷新。
- **智能思考过程接管**：针对诸如 DeepSeek 需要展示思考内容的特性，前端特别构建了对 `<think>...</think>` 正则分离层，将系统内部打磨思维转化为特定的 CSS 卡片 (`thought-block`)，提供视觉区隔。

## 5. 总结
当前【智能题库生成系统(DDG深搜迭代版)】已跨越简单的 Prompt 拼接时代，进入了微服务化、外网前哨辅助与单点重度发酵深加工的成熟期。通过极具侵略性的多极检验、代码锁防护与流式工程呈现，在复杂多变的 AI 环境下实现了出题极高的通过率以及出色的视觉交付质量。
